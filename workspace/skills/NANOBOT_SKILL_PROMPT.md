# Промпт: написание навыков для Nanobot

Используй этот промпт когда просишь агента написать или доработать навык для Nanobot.

---

Ты пишешь навык (skill) для агента **Nanobot** (https://github.com/HKUDS/nanobot).

## Что такое Nanobot

Ультралёгкий Python-агент на базе LLM. Управляется через `~/.nanobot/config.json`. Запускается как CLI (`nanobot agent`) или как gateway-сервис (Telegram, Discord, Slack и др.).

## Структура данных

```
~/.nanobot/                        # Корень (Windows: C:\Users\Сергей\.nanobot\)
├── config.json                    # ⚠️ API-ключи, токены каналов — СЕКРЕТЫ
├── workspace/
│   ├── SOUL.md                    # Личность агента (system prompt)
│   ├── AGENTS.md                  # Инструкции агентов
│   ├── USER.md                    # Информация о пользователе
│   ├── TOOLS.md                   # Документация инструментов
│   ├── HEARTBEAT.md               # Задачи heartbeat
│   ├── memory/
│   │   └── MEMORY.md              # Долгосрочная память
│   ├── skills/                    # Навыки — ПАПКИ, не файлы
│   │   └── имя_навыка/
│   │       └── SKILL.md           # Единственный обязательный файл навыка
│   └── sessions/                  # Сессии переписки (личные данные)
├── cron/
│   └── jobs.json                  # Крон-задания
└── history/
    └── cli_history                # История CLI
```

## Как устроен навык (skill)

Навык — это **папка** в `~/.nanobot/workspace/skills/` с файлом `SKILL.md` внутри.  
Никаких `.py`-файлов навыка не существует. Нет `main()`, нет `argparse`, нет импортов между навыками.

`SKILL.md` — это **инструкция для агента**, написанная на человеческом языке плюс готовые к выполнению команды.

### Обязательный заголовок SKILL.md

```markdown
---
name: имя-навыка
description: Одно предложение — что делает навык. Агент использует это для автовыбора навыка.
metadata: {"nanobot":{"emoji":"🔧","requires":{"bins":["python3","git"],"env":[]}}}
---
```

## Как агент выполняет навык

1. Читает `SKILL.md` через `read_file`
2. Следует инструкциям из файла
3. Выполняет команды через инструмент `exec`
4. Читает файлы через `read_file`, пишет через `write_file`, смотрит директории через `list_dir`

**Агент НЕ запускает `.py`-файлы навыка напрямую.** Он выполняет команды из инструкций.

## Правила написания SKILL.md

### ✅ Правильно

- Команды написаны как `python3 -c "..."` — полностью самодостаточные, без внешних импортов
- Каждый блок кода работает независимо, агент может скопировать и выполнить его через `exec`
- Используется `pathlib.Path.home() / '.nanobot'` — не хардкодить пути
- Переменные между шагами передаются через `print()` → агент читает вывод и подставляет в следующую команду
- Инструкции чёткие: "агент выполняет X, затем Y, при ошибке Z"
- Есть таблица принятия решений: какой запрос → какое действие

### ❌ Неправильно

- Многострочные Python-скрипты с `class`, `argparse`, `if __name__ == '__main__'` — агент пытается найти этот файл и зависает
- `from check_secrets import ...` — нельзя импортировать между блоками SKILL.md
- Хардкод путей типа `C:\Users\Сергей\` — использовать `pathlib`
- `bash`-команды с `tar`, `rsync`, `find` в стиле Linux на Windows — использовать Python
- Команды без проверки существования файлов/директорий

## Среда выполнения

Текущая: **Windows 11**, путь `C:\Users\Сергей\.nanobot\`.  
Планируется: переход на **WSL2**.

Определение среды в Python:
```python
import sys, pathlib
env = "windows" if sys.platform == "win32" else \
      "wsl2" if "microsoft" in pathlib.Path("/proc/version").read_text().lower() else "linux"
```

`pathlib.Path.home()` корректно обрабатывает кириллику в имени пользователя на Windows.  
Все команды должны использовать Python (`python3 -c`) вместо bash-утилит — это обеспечивает кросс-платформенность.

## Работа с GitHub

Если навык пишет в GitHub:
1. Клонировать репозиторий в `tempfile.mkdtemp(prefix='nanobot_')` 
2. Внести изменения
3. Сделать коммит и push
4. Удалить временную папку через `shutil.rmtree`

URL репозитория — спросить у пользователя если неизвестен. Для авторизации — использовать GitHub-скилл.

**Никогда не публиковать в GitHub:** `config.json`, `sessions/`, `history/`, любые файлы с API-ключами или токенами.
